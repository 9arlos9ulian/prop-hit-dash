# prop-hit / docs
hit-model v0.5 · personal use
MLB batter hit-probability model + decision dashboard · not an autobetting system
⎇ github.com/9arlos9ulian/prop-hit
overview
Predicts P(batter records ≥1 hit | game) for every starter in tonight's lineup. Output is a calibrated probability — not a pick, not a recommendation. The model surfaces edge; you decide whether the market price is worth it.

validation requirements
Two checks must pass on a held-out period before treating output as betable:
01
Calibration — when model says 70%, actual hit rate should be ~70%. Brier score 0.226 on 45-day holdout. Check via History → Calibration Buckets.
02
Closing-line value — model edges must beat the de-vigged closing line. pct_beat_close is the real signal. ROI on small samples is noise.
views
/tonight
Full batter pool sorted by P(hit). Card shows streak dots (L5), HOT/COLD badge if streak >5 games, HH badge, BvP or career split vs pitcher hand. Filters: lineup slot, game, hot streak, confirmed lineups. Bet slip via FAB.
/no-hit
Inverted view — sorted by highest P(no hit). COLD/radioactive badge for cold streaks. Same tracker and filters as /tonight.
/slate
Game-level view. Pitcher matchup ratings, team offensive ratings, final scores, game times. Sort by time or rating.
/history
Logged picks vs outcomes. Daily scorecard, Brier score, calibration bucket chart. Settled nightly via settle.py.
